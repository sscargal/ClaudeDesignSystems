Fluent Design System

Button

Buttons trigger actions or navigate users through a flow. Fluent 2 offers five visual variants and three size options, each with consistent interaction states.

Variants
Choose the variant that matches the action's hierarchy. Use one Accent per context, Standard for secondary actions, and Subtle or Transparent for tertiary.
All variants — light background
All variants — dark background
Variant Class Use when Background
.btn-accent Primary call-to-action. One per view. --fill-accent-default (#0078D4)
.btn-standard Secondary action alongside Accent. White / Surface fill + border
.btn-subtle Tertiary, low-priority actions. Transparent → fill-subtle on hover
.btn-outline Balanced weight between Subtle and Standard. Transparent + border
.btn-transparent Minimal UI — toolbars, inline actions. Fully transparent
.btn-danger Destructive actions (delete, remove). --color-error (#C42B1C)
